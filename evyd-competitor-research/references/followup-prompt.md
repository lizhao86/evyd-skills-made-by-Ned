# 追问模式（Follow-up Mode）Agent Prompt 模板

## 首次追问 - 生成 3 个深挖问题

当调研报告已完成并输出到飞书文档后，执行以下任务：

### 任务
基于已生成的调研报告内容，提出 3 个值得深挖的问题。

### 问题设计原则
1. **问题 1 - 商业模式深挖**：
   - 付费方细节（如：政府预算来源、医院采购决策流程）
   - 定价策略（如：是否有公开价目表、折扣策略）
   - 收入结构（如：主要收入来源占比）

2. **问题 2 - 技术/产品深挖**：
   - 技术架构细节（如：是否支持多云部署、数据存储方案）
   - 产品功能细节（如：具体 workflow、AI 能力细节）
   - 集成深度（如：与 Epic/Cerner 的集成方式）

3. **问题 3 - 市场/竞争深挖**：
   - 竞争策略（如：与主要竞品的差异化打法）
   - 市场动态（如：近期重大合作、产品发布）
   - 客户反馈（如：用户评价、案例研究）

### 输出格式（对话回复，不出现在文档中）
```
🦐 调研完成！文档已保存至：[飞书文档链接]

基于现有信息，我建议深挖以下 3 个问题：

1. **【商业模式】** {具体问题}
   → 可深挖：{简要说明价值}

2. **【技术细节】** {具体问题}
   → 可深挖：{简要说明价值}

3. **【竞争策略】** {具体问题}
   → 可深挖：{简要说明价值}

你可以：
- 回复序号（如"1"或"深挖第2个"）继续调研
- 直接提出新问题
- 说"结束"完成本次调研
```

**注意**：以上内容只在对话中展示，不写入飞书文档。

## 持续追问 - 深挖用户问题

### 任务
对用户提出的问题进行深度调研，并输出回答。

### 执行步骤
1. 使用 `web_search` 检索用户问题的答案（至少 3-5 次搜索）
2. 整理回答，标注证据来源
3. **询问用户是否追加到原文档**：
   ```
   是否将本次回答追加到原文档 `For [user] - Research - {竞品名} - {日期}` 中？
   
   回复：
   - "是" / "追加" / "同意" → 追加到原文档
   - "否" / "不用" → 仅作为对话回复，不修改文档
   ```

### 输出格式
```markdown
### 追问回答：{用户问题}

#### 核心发现
{回答内容，使用列表和段落，不要用表格}

#### 证据

**证据 1**
- 结论点：
- 来源：
- 可信度：

**证据 2**
- 结论点：
- 来源：
- 可信度：

**证据 3**
- 结论点：
- 来源：
- 可信度：

#### 链接
```
1) ...
```
```

## 追加到原文档

### 触发条件
用户同意追加（回复"是"/"追加"/"同意"等）

### 执行步骤
1. 使用 `feishu_doc` 的 `read` 或 `list_blocks` 获取原文档内容
2. 默认**在文档末尾新建追问章节**，不要把追问内容塞回原有章节内部
3. 使用 `feishu_doc` 的 `append` 追加；如失败，再拆成更短的多次 `append`
4. **即使拆分多次 append，也必须保持同样的结构**，不能退化成一整段正文+链接混排

### 强制格式规则（必须执行）

#### 标题格式（一级标题）
```markdown
# {问题摘要} - {YYYY-MM-DD HH:mm}
```

**要求：**
- 必须是一级标题 `#`
- 不要加“追问 - ”前缀，直接用问题摘要
- 标题中必须带时间戳

#### 正文格式
```markdown
正文：
- 结论 1
- 结论 2
- 结论 3
```

**要求：**
- 正文单独成块
- 优先用短段落或 bullet point
- **禁止**把“正文 + 所有链接”挤成一个长段落
- 如果内容过长，优先删冗余，不要破坏结构

#### 证据链接格式
```markdown
证据链接：
- https://...
- https://...
- https://...
```

**要求：**
- 必须单独成块
- 必须使用 bullet list
- 只放链接，不要把大段解释塞进链接区
- 链接数量优先控制在 3-8 条最关键来源

### 推荐追加模板（默认使用）
```markdown
# {问题摘要} - {YYYY-MM-DD HH:mm}

正文：
- 结论 1
- 结论 2
- 结论 3

证据链接：
- https://...
- https://...
- https://...
```

### append 失败时的回退策略（必须遵守）
如果一次 `append` 失败：
1. 先保留标题不变
2. 缩短正文，只保留最关键 3-5 条结论
3. 链接压缩为最关键 3-5 条
4. 分两次追加也可以，但顺序必须是：
   - 第一次：标题 + 正文
   - 第二次：`证据链接：` + bullet list
5. **绝不能**为了省事改成一整段长文本

### 追加完成后的回复
```
🦐 已追加到原文档！

更新时间戳：{YYYY-MM-DD HH:mm}
文档链接：[飞书文档链接]

还有其他想深挖的问题吗？
```

## 追问模式结束

当用户说以下任意一种表达时，结束追问模式：
- "结束"
- "完成"
- "不用了"
- "先这样"
- "谢谢"

结束语：
```
🦐 调研完成！文档链接：[飞书文档链接]

如需继续补充，随时说"追问"或提出新问题。
```
